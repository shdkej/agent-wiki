# 현실 속에서 이상을 키우기

- 원본: `source/shdkej-content/Human/Reality.md`
- Source note updated: `2025-07-23`
- 상태: 안정 디벨롭
- 역할: 원본 만다라트 노드와 1:1 대응하는 Human 카테고리 관리 노트

## 핵심 논지

이 노드는 현실주의를 권하는 메모가 아니다. **이상, 이성, 감성, 추상, 계획**이 실제 삶의 마찰과 만나면 무엇이 살아남고 무엇이 무너지는지 시험하는 Human 노드다. 현실은 이상의 반대편이 아니라, 이상이 장식인지 작동 가능한 방향인지 판별하는 검증장에 가깝다.

## 원문 이동 경로

원문은 생활 메모, 예술사, 추상화, 소프트웨어 설계가 섞여 있지만 이동 방향은 비교적 분명하다.

| 원문 층 | 대표 장면 | Reality에서의 의미 |
| --- | --- | --- |
| 생활 마찰 | 자취의 로망, 수납 위치, 출근 시간, 입맛 없을 때 운동과 다이어트 | 맞는 말이 실제 몸·시간·동선에 들어갈 때 어떻게 다른 처방이 되는지 본다 |
| 이성·감성 | 모더니즘, 히틀러, 포스트모더니즘, AI 이후 인간의 정서와 관계성 | 논리와 질서만으로는 현실을 다룰 수 없고 감정·관계·맥락이 함께 작동한다 |
| 이상·현실의 역사 | 플라톤/아리스토텔레스, 다빈치/미켈란젤로, 르네상스/바로크 | 보이지 않는 이상과 눈앞의 현실 중 무엇을 더 본질로 볼지 오래된 비교 거울로 삼는다 |
| 실제·상상 | 실제하지만 알지 못했던 것, 상상하지만 실제하지 못하는 것, 아이디어와 구현의 간극 | 현실을 이미 주어진 제약이 아니라 발견되지 않은 실제와 구현되지 않은 상상 사이의 검증면으로 본다 |
| 추상·설계 | 카테고리, Unix 입출력, Go duck typing, 문자열 라이브러리, workflow, 메뉴얼 | 좋은 추상은 평상시 복잡함을 줄이되 문제 상황에서 아래층으로 내려갈 통로를 남긴다 |

이 배열 때문에 `Reality`는 잡다한 메모 모음이 아니라 **생활의 마찰 -> 사상적 비교 -> 설계/구현 재번역**의 루프를 가진 카테고리로 읽힌다.

## 세 축으로 압축한 현실 감각

### 1. 이상과 실행 저항

원문은 이상을 버리라고 하지 않는다. 다만 현실적인 문제를 해결하지 못하는 이상은 무력하다고 본다.

- 자취를 하면 집을 잘 꾸미고 밥도 잘 차려 먹을 줄 알았지만, 실제로는 귀찮음과 동선이 더 크게 작동한다.
- 수납함에 넣어 두면 깔끔하지만, 자주 쓰는 물건은 집기 편한 위치에 그냥 놓게 된다.
- 출근 시간에 늦지 않으려면 `일찍 일어나면 된다`는 정답만으로 부족하고, 전날 밤 동선·수면·알람 조건이 함께 바뀌어야 한다.
- 임신·출산 이후에도 커리어를 그대로 이어가겠다는 계획은 실제 생애주기 변화를 만나면 다른 세계로 들어간다.
- 입맛이 없을 때 운동과 다이어트를 붙이면 이론상 식욕이 돌아올 수 있지만, 실제 상태가 너무 낮으면 부담이 먼저 커질 수 있다.

따라서 현실의 첫 질문은 `왜 의지가 약한가`가 아니다. **귀찮음, 시간 제한, 복잡도, 몸 상태, 생애주기 변화** 중 어떤 마찰이 실제 실행을 막고 있는지 분류하는 것이다.

### 2. 이성·감성·예술사의 비교 거울

원문 중간의 예술사 메모는 교양 부록이 아니다. 생활의 마찰을 더 큰 사고 전통 속에 비춰 보는 비교 거울이다.

- 플라톤은 이데아를, 아리스토텔레스는 자연을 더 중시한다.
- 다빈치는 자연 모방을, 미켈란젤로는 내면에서 발견한 형상을 더 강하게 떠올리게 한다.
- 르네상스는 질서·비례·척도와 자연 모방을, 바로크는 감정과 불명료함의 힘을 더 드러낸다.
- 모더니즘은 이성과 질서를 크게 믿었지만, 이성도 폭력적으로 사용될 수 있다는 한계를 드러냈다.
- AI가 형식적 이성의 일부를 더 잘하게 될수록 인간에게는 감정, 관계, 맥락 감각이 더 중요해질 수 있다.

이 축은 `현실을 보라`는 말을 단순한 현실 순응으로 낮추지 않는다. 오히려 무엇을 본질로 볼지, 눈앞의 구체를 얼마나 신뢰할지, 보이지 않는 질서를 어디까지 믿을지 묻는다.

### 3. 추상과 사용 현실

후반부의 추상화 메모는 개발 부록이 아니라 Reality의 핵심 축이다. 추상은 본질을 남겨 여러 경우를 다루게 하지만, 실제 조건으로 되돌아갈 통로가 없으면 현실을 더 멀게 만든다.

- `관객은 추상이다`: 창작자는 실제 개별 관객을 직접 만질 수 없으므로 추상된 상대를 상정해 표현과 해설을 설계한다.
- 현대미술 해설은 작품과 관객 사이의 접속면이지만, 해설이 작품을 대신 꾸미기 시작하면 현실의 결과물보다 설명 언어가 더 큰 권위를 갖는다.
- 카테고리도 머릿속 분류로 끝나지 않는다. `think`를 어디에 둘지, `review`를 Reading 아래에 둘지 같은 질문은 실제 재호출 동선 앞에서 다시 시험된다.
- 문자열 라이브러리처럼 좋은 추상은 입문 비용을 줄인다. 사용자는 0과 1을 직접 다루지 않아도 문자를 사용할 수 있다.
- 반대로 workflow가 실제 실행 명령과 옵션을 가리면, 문제가 생겼을 때 원인과 복구 경로로 내려가기 어렵다.
- 자주 일어나지 않는 사건일수록 읽고 바로 실행할 수 있는 메뉴얼이 필요하다.

그래서 이 노드에서 좋은 추상은 복잡함을 영원히 숨기는 포장이 아니다. **평상시에는 쉽게 쓰게 하고, 예외 상황에서는 실제 실행 내용·옵션·실패 조건·복구 절차로 내려가게 하는 구조**다.

## 현실을 다룰 때의 오판

| 오판 | 왜 위험한가 | 다시 물을 질문 |
| --- | --- | --- |
| 귀찮음과 구조적 제약을 섞는다 | 구조 문제를 의지 문제로 오독하면 자책만 늘고, 회피를 모두 현실 탓으로 돌리면 조정 지점을 놓친다 | 지금 막는 것은 동선, 시간, 체력, 감정, 역할 변화 중 무엇인가 |
| 현실을 핑계로 방향까지 포기한다 | 이상이 없으면 방향이 사라지고, 이상만 있으면 현재를 못 넘긴다 | 이상을 버릴지, 구현 단위를 낮출지, 환경을 바꿀지 구분했는가 |
| 상상과 구현 가능성을 한쪽만 본다 | 이미 구현된 것만 현실로 보면 새로운 가능성을 못 보고, 상상만 붙잡으면 실제 조건을 얻지 못한다 | 발견되지 않은 실제와 구현 조건을 얻지 못한 상상을 나눠 보았는가 |
| 추상을 정답으로 고정한다 | 분류·라이브러리·workflow가 실제 사용과 복구 상황에서 오히려 현실을 가릴 수 있다 | 문제가 생겼을 때 아래층으로 내려갈 관측 표면이 있는가 |

## 재사용 프레임

1. 내가 세운 이상, 개념, 설계 의도를 먼저 적는다.
2. 실제로 부딪히는 마찰을 생활 장면의 언어로 쓴다.
3. 그 마찰이 귀찮음·회피인지, 시간·몸·환경·생애주기 같은 구조적 제약인지 나눈다.
4. 이미 존재하지만 보지 못한 현실과, 상상했지만 구현 조건이 부족한 가능성을 분리한다.
5. 이상을 버릴지, 구현 방식을 바꿀지, 환경을 바꿀지 결정한다.
6. 추상이나 메뉴얼이 문제 상황에서 실제 실행 경로로 돌아가게 해 주는지 확인한다.

이 프레임 덕분에 `Reality`는 냉소적 현실주의가 아니라 **이상이 살아남는 조건을 찾는 조정 노드**로 남는다.

## 후속 흡수 기준

- 새 생활 사례는 단순 일화로 추가하지 말고 `실행 저항` 표의 마찰 유형을 더 정확히 대체할 때만 본문에 흡수한다.
- 새 예술·철학 메모는 지식 목록으로 늘리지 말고, 이상/현실 또는 이성/감성 비교축을 더 선명하게 만들 때만 추가한다.
- 새 개발·추상화 사례는 추상 수준 논쟁보다 `아래층으로 내려갈 통로`가 있는지 판정하는 예시일 때만 남긴다.
- 패스별 편집 이력은 본문에 쌓지 않고 `agent-wiki/log.md`에만 기록한다.
