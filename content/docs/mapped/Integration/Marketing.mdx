# 마케팅

이 페이지는 `source/shdkej-content/Integration/Marketing.md`와 1:1로 대응한다. 개별 경로 관리는 [[../../maintenance/source-mapping]]에서 확인한다.

## 한 줄 요약
Marketing은 광고 채널 관리보다 `사용자 접점 설계 -> 반응 관찰 -> 첫 가치 확인 -> 전환 설득 -> 제품/CRM 조정`을 잇는 Integration 노드다.

## 만다라트 위치
`Integration/Marketing`은 여러 실행 표면을 하나의 사용자 흐름으로 묶는 노드다. 같은 폴더의 [[Business]]가 사업 판단과 조직적 검증을 넓게 보고, [[Creator]]가 표현자와 보이는 인상을 다루며, [[../deep-knowledge/product]]가 기능·지표·제품 실험의 내부 루프를 깊게 본다면, Marketing은 그 앞뒤를 연결한다.

이 페이지의 핵심 질문은 `어떤 사람이 어떤 말과 표면을 보고 들어오며, 어디서 멈추고, 무엇을 느껴야 결제하거나 계속 쓰는가`다. 그래서 원문에 CRM, 제품, UX 라이팅, SNS가 함께 들어 있는 것은 잡항이 아니라 같은 사용자 이동 경로의 다른 접점이다.

## 원문 구조
- 개요: SNS, 마케팅, UX 라이팅
- 마케팅과 CRM과 제품
- 마케팅의 4P
- 마케팅은 어디서부터 시작하나
- 마케팅 기본 마인드셋
- UX 라이팅
- SNS

## 중심 모델

### 1. 사용자 흐름
원문은 `고객 유입은 마케팅`, `고객 유지는 CRM`, `고객 이탈 방지는 제품`이라고 나누지만, 곧바로 `전환`이라는 빠진 층을 추가한다. 이 구분은 부서 정의가 아니라 사용자가 `알게 됨 -> 첫 가치를 확인함 -> 구매함 -> 남음 -> 떠나지 않음`으로 이동할 때 어느 접점이 비어 있는지 보는 지도다.

| 구간 | 원문 표현 | 봐야 할 질문 |
| --- | --- | --- |
| 유입 | 고객 유입, SNS, 채널별 글 | 누가 어떤 표면에서 처음 들어오는가? |
| 활성화 | 무료 활성사용자 | 들어온 사람이 첫 가치를 실제로 확인하고, 다음 가치로 다시 이동하는가? |
| 전환 | 왜, 언제 구매하지를 설득 | 지금 돈을 내야 하는 이유가 체감되는가? |
| 유지 | CRM, 고객 플래그 | 다음 접점에서 덜 추측하게 할 맥락이 남는가? |
| 이탈 방지 | 제품이 없으면 불편 | 제품 경험 자체가 떠나기 어렵게 만드는가? |

### 2. 사람 해석
마케팅은 채널 선택보다 `핵심 고객 타겟팅`에서 시작한다. 원문은 무수한 대상을 한 번에 상대하기보다 한 명 한 명을 상상해서 넓혀 가라고 적는다. 이는 대중을 빨리 모으는 기술보다, 먼저 어떤 사람이 왜 움직이지 않는지 읽는 능력을 중시한다는 뜻이다.

소비자 관점도 단순 공감이 아니다. 원문은 소비자가 모르는 불편까지 느끼고, 소비자가 듣고 싶어 하는 얘기를 알아야 한다고 말한다. 즉 마케팅은 브랜드가 말하고 싶은 문장을 앞세우는 일이 아니라, 사용자가 아직 언어화하지 못한 불편·망설임·기대를 먼저 해석하는 일이다.

### 3. 전환 설계
원문의 강한 문장은 `전환이라고 하는 왜, 언제 이 제품을 구매하지를 설득하는 부분`이다. 유입과 유지 사이에 이 층을 따로 세워야 마케팅이 노출 수나 기능 설명으로 평평해지지 않는다.

무료 사용자를 유료 고객으로 옮기려면 제품이 없어졌을 때 불편해야 한다. 이때 무료/유료 경계는 단순 가격표가 아니다. 무료에는 시간, 동시 개수, 수작업 같은 체험 가능한 마찰이 남고, 유료에는 자동화와 편의가 붙는다. 전환은 어떤 불편을 남기고 어떤 불편을 제거할지 정하는 제품 설계 문제이기도 하다.

다만 전환은 결제 직전의 설득만 뜻하지 않는다. 무료 활성사용자 메모를 같이 보면, 먼저 `첫 이득 -> 다시 쓸 이유`가 이어져야 유료 설득이 자연스러워진다. 그 전에 멈추게 하는 화면·입력·기다림은 구매 설득 실패가 아니라 가치에 도착하기 전의 경로 실패로 따로 본다.

### 4. 표현과 채널
UX 라이팅 메모의 `마케팅 퍼널에서 사용자 질문 목록을 좌에서 우로 가는 한판의 그림`은 이 노트의 실행 손잡이다. 좋은 문구는 브랜드 설명을 순서대로 늘어놓는 것이 아니라, 사용자가 각 단계에서 실제로 묻는 질문에 답해야 한다.

메일침프식 톤 문장도 장식이 아니라 경계 조건이다.

- 재미있지만 유치하지는 않다.
- 캐주얼하지만 조잡하지는 않다.
- 전문적이지만 잘난 척하지 않는다.

개성이 있어야 애정받는 브랜드가 될 수 있지만, 개성이 사용자 질문을 가리거나 신뢰를 떨어뜨리면 전환을 방해한다. 그래서 톤 가이드는 더 튀라는 주문보다 `관심은 만들되 불안을 키우지 않는 수위 조절`에 가깝다.

이 원칙은 채널 확장보다 응답 표면 점검에 가깝다. 제품 밖 설명은 대상, 상황, 차이, 한계를 보여 줄 때만 원문의 `전문적이지만 잘난 척하지 않는다`는 경계와 맞는다.

### 5. 지속성과 재방문
SNS 메모는 게시 빈도 팁보다 `꾸준히 계속 하는 사람이라는 인상`, `자기의 표현을 하는 사람`, `그의 향후에 궁금증을 갖게 하는 것`을 남기는 표면이다. 한 번의 글은 제품 설명이지만, 반복되는 글은 방향성과 지속성을 증명한다. 이때 지속성은 매일 끊기지 않는 압박보다, 사용자가 한동안 멈췄다가도 다시 궁금해하고 돌아올 이유가 남는 상태에 가깝다.

오픈헌트, 지피터스, 쓰레드, 인스타, 긱뉴스에 같은 글을 돌리지 말라는 메모도 같은 맥락이다. 매체별 글은 달라야 하지만, 달라지는 것은 핵심 제안이 아니라 번역 방식이다. 어떤 표면에서는 기술적 소개가 먼저 먹히고, 어떤 표면에서는 이미지·짧은 사용 필요성·상품 소개·정보 공유가 더 잘 작동한다.

## 실행 판단판
아래 표는 원문 단서를 `유입 / 활성화 / 전환 / 유지 / 재방문` 접점으로 다시 들어가기 위한 판정판이다. 보조 자료는 독립 주제가 아니라 어느 접점을 실제로 선명하게 하는지 확인한 뒤에만 흡수한다.

| 실행 표면 | 원문 단서 | 유지할 판단 | 보조 판정 |
| --- | --- | --- | --- |
| 핵심 고객 | 한 명 한 명 상상 | 대중 확산 전에 한 사람의 불편과 언어를 선명하게 잡는다. | 누구에게 먼저 약속해도 되는지 좁힌다. |
| 유입 분석 | 유입 경로 3가지 분리 | 채널을 합산하지 말고 어디서 어떤 기대를 들고 왔는지 나눈다. | 작은 표본은 성과보다 반응 언어와 대체재 단서로 읽는다. |
| 고객 플래그 | 유입될 때 플래그 | 사람을 고정 분류하기보다 다음 대화와 CRM 문장을 덜 추측하게 한다. | 실사용자, 테스트, 자동화 트래픽을 섞지 않아야 반응 언어가 흐려지지 않는다. |
| 무료/유료 | 시간·동시 개수·수작업 / 자동화 | 무료는 문제 인지, 유료는 수작업 제거와 속도 회복으로 설계한다. | 결제 이전에 제품 안에서 가치가 체감되는지 먼저 본다. |
| 첫 가치 | 무료 활성사용자, 좌에서 우로 한판의 그림 | 첫 행동 뒤 사용자가 기대한 결과를 확인하고 다시 쓸 이유까지 얻는지 본다. | 다음 가치로 이어질 행동이 없으면 활성사용자를 전환 후보로 과대해석하지 않는다. |
| 도달 전 마찰 | 유입 경로 3가지, 좌에서 우로 한판의 그림 | 첫 가치 이전의 필드, 화면, 결정, 대기가 사용자가 온 이유를 늦추는지 점검한다. | 반복 클릭, 입력 후 멈춤, 결과 전 이탈은 전환 실패 전에 경로 실패로 읽는다. |
| 퍼널 질문 | 좌에서 우로 한판의 그림 | 단계별 사용자 의심을 화면·메일·랜딩 문장으로 옮긴다. | 4P나 AARRR도 암기 프레임보다 멈춤 구간을 찾는 진단판으로 쓴다. |
| 채널 글쓰기 | 각 매체별 글은 다름 | 같은 제안을 채널 문법에 맞게 다시 번역한다. | 표현의 결은 신뢰, 친근함, 수준감, 기대를 조절하는 장치다. |
| 응답 표면 | 소비자가 듣고 싶어 하는 얘기, 좌에서 우로 한판의 그림 | 제품 밖 공개 설명이 누구의 어떤 망설임을 줄이는지 점검한다. | 대상·상황·차이·한계는 그 망설임을 줄일 때만 남긴다. |
| SNS 운영 | 꾸준함, 자기표현, 향후 궁금증 | 단기 홍보보다 시간에 걸친 신뢰와 다음 기대를 만든다. | 연속성 지표보다 멈춘 뒤에도 다시 떠올릴 표현·제품 이유가 남는지 본다. |

## 인접 노드
인접 노드는 링크 수를 늘리기보다 Marketing 안에서 맡는 역할로 묶어 둔다.

| 역할 | 연결 노드 | Marketing에서의 의미 |
| --- | --- | --- |
| 사업 검증 | [[Business]] | 고객 창출과 운영 검증의 넓은 바깥 원 중, 사람을 움직이게 만드는 접점 설계만 좁혀 본다. |
| 표현 표면 | [[Creator]], [[Design]], [[../Communication/Blogging]], [[../Human/Readability]] | 표현자 인상, 브랜드 톤, 채널별 글쓰기, 읽기 쉬운 문장이 유입과 전환 설득으로 번역되는 지점을 본다. |
| 제품 루프 | [[../deep-knowledge/product]] | 제품 경험이 잔존과 이탈을 가르는 내부 루프로 들어가기 전 기대와 언어를 보존한다. |
| 반응 회수 | [[../Communication/Feedback]] | 유입 이후 반응을 해석해 다음 메시지, CRM 문장, 제품 행동으로 되돌리는 회로와 이어진다. |

## 선호 신호
저자는 마케팅을 화려한 홍보 기술보다 사람 이해, 전환 설계, 제품 경험, 말투 조정, 채널별 인상 관리가 결합된 운영 문제로 보는 경향이 있다. 대규모 노출보다 핵심 고객의 선명한 상상, 무료에서 유료로 넘어가는 설득 구조, 브랜드 톤의 미세 조정을 더 중요한 실무 감각으로 둔다.

후속 자료는 새 이론으로 따로 붙이기보다 `실행 판단판`의 기존 행 하나를 더 선명하게 만들 때만 흡수한다. 특히 onboarding, activation, retention, AEO/GEO 같은 외부 렌즈는 원문에 있는 유입 경로, 고객 플래그, 첫 가치, 도달 전 마찰, 채널별 글쓰기, SNS 운영 중 어디를 실제로 대체하거나 구체화하는지 먼저 판정한다. 첫 가치 자료를 참고하더라도 새 지표 섹션을 만들기보다 `결과 확인`, `다음 가치`, `도착 전 경로 실패`의 구분을 흐리지 않는 데만 쓴다.
