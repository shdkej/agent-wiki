# README

- 원본: `source/shdkej-content/README.md`
- 상태: 점진 심화됨
- 역할: 원본 만다라트 레이어 자체의 경계와 취급 방식을 설명하는 루트 카테고리 페이지

## 한눈에 보는 요약
이 노드는 사용자의 원본 노트 내용을 직접 요약하는 페이지라기보다, `source/shdkej-content/` 전체를 어떻게 다뤄야 하는지 알려 주는 경계 표식이다. 핵심은 원본 레이어를 작업 재료로 마음대로 고치는 공간이 아니라, 주제 경계와 폴더 구조를 보존한 채 `agent-wiki/`에서 해석·비교·합성을 수행해야 하는 1차 소스로 보는 것이다.

## 핵심 논지
`source/shdkej-content/`는 임의 문서 저장소가 아니라 사용자의 원본 content/노트에서 온 1차 소스 레이어다. 따라서 이 레이어의 가치는 개별 문장뿐 아니라 폴더 구조, 파일 단위, 주제 경계, 수정 절차에 함께 있다. 좋은 에이전트 정리는 원본을 깨끗하게 덮어쓰는 일이 아니라, 원본은 그대로 두고 대응 위키 페이지에서 해석 가능한 구조를 만드는 일이다.

이 README는 다른 카테고리처럼 특정 주제를 길게 설명하지 않는다. 대신 전체 만다라트 원본을 다루는 `출입 규칙`에 가깝다. 이 노드가 약하면 각 mapped 노트는 내용 요약에는 성공해도, 원본의 거친 순서·비어 있는 항목·중복 질문을 왜 보존해야 하는지 잊기 쉽다.

## 왜 이 노드가 중요한가
- 1:1 mapped 원칙의 기준점이다. 다른 노트들이 각 카테고리 내용을 담는다면, 이 노드는 그 카테고리들을 어떻게 취급할지 정한다.
- [[Portfolio]], [[CHANGELOG]], [[Meta/Collection]] 같은 루트·메타 노드를 읽을 때, 원본의 불완전함이나 거친 기록을 함부로 삭제하지 말아야 하는 이유를 제공한다.
- `source/external-links/`와 달리 이 레이어가 사용자의 원본 만다라트 구조에서 온 1차 소스임을 구분하게 만든다.
- 이후 drift 점검, human-review 복사, source 수정 금지 같은 운영 절차의 근거가 된다.

## 원본에서 직접 보이는 구조

### 1. 레이어 선언
원문 첫 문장은 `source/shdkej-content/`가 사용자의 원본 content/노트에서 온 1차 소스라고 못박는다. 여기서 중요한 단어는 `원본`과 `1차 소스`다. 에이전트가 이 폴더를 정리 대상 폴더로 보면 안 되고, 위키가 참조해야 할 출처 레이어로 봐야 한다.

### 2. 읽기 전용 원칙
규칙의 첫 항목은 본문을 기본적으로 읽기 전용으로 둔다고 말한다. 이것은 단순한 파일 보호가 아니라 해석 책임의 분리다.

- 원본에는 사용자의 실제 기록 습관, 빈칸, 중복, 덜 익은 질문이 남아 있다.
- agent-wiki에는 그 재료를 읽어낸 구조, 요약, 연결, 판단이 남는다.
- 원본이 불편해 보여도 에이전트가 먼저 손대면, 나중에 선호 변화와 실제 기록 흔적을 구분하기 어려워진다.

### 3. 폴더 구조와 주제 경계 보존
원문은 폴더 구조와 주제 경계를 유지하라고 한다. 이는 이 랩에서 각 markdown 노트를 8x8 만다라트식 카테고리 노드로 보는 규칙과 바로 연결된다.

즉 파일은 단순한 글 조각이 아니라 다음 정보를 함께 가진다.

| 요소 | 읽는 법 |
| --- | --- |
| 폴더 위치 | 이 노트가 어느 생활·기술·사고 영역에 놓이는지 보여 준다. |
| 파일 단위 | 사용자가 하나의 카테고리로 묶어 본 사고 단위다. |
| 빈칸과 미완성 | 삭제 대상이 아니라 아직 질문이 덜 자란 상태 신호일 수 있다. |
| 중복과 거친 표현 | 정리 실패가 아니라 어떤 질문이 반복해서 돌아오는지 보여 주는 흔적이다. |

## 만다라트 구조에서의 위치

루트 README는 만다라트의 한 칸이라기보다, 격자 전체를 어떻게 만질지 정하는 표면이다. 그래서 이 노드를 읽을 때는 `무슨 내용이 들어 있는가`보다 `내용을 어디에서 바꿀 수 있는가`를 먼저 봐야 한다.

- `source/shdkej-content/`의 각 markdown 노트는 원본 카테고리 노드다.
- `agent-wiki/content/docs/mapped/`의 각 대응 문서는 그 노드를 해석한 컴파일 결과다.
- 루트 README는 두 레이어가 서로 섞이지 않게 하는 경계 문서다.
- 이 경계가 유지되어야 이후 external-links, syntheses, concepts가 원본 만다라트의 의미를 덮어쓰지 않는다.

## 운영 해석

### source와 agent-wiki는 역할이 다르다
원문은 정리, 해석, 비교, 합성을 `agent-wiki/`에서 수행하라고 한다. 따라서 mapped 노트는 원본을 예쁘게 복사한 두 번째 원본이 아니라, 원본을 다시 사용할 수 있게 만드는 컴파일 레이어다.

- source: 원래 기록의 위치, 질감, 불완전함을 보존한다.
- mapped: 원본과 1:1로 대응하며 핵심 구조를 읽어 낸다.
- syntheses/concepts: 여러 mapped 노드에서 반복되는 판단을 장기 지식으로 올린다.
- log: 어떤 노드를 언제 어떤 관점으로 갱신했는지 남긴다.

### 원본 수정은 별도 사건이다
원문은 원본 수정이 필요하면 변경 이유와 영향을 분리해서 기록하라고 한다. 이것은 사용자의 직접 수정이 들어왔을 때 `human-reviews/`에 복사하고 선호 차이를 문서화하라는 운영 규칙과 같은 방향이다.

중요한 점은 원본과 에이전트 해석을 섞지 않는 것이다. 사용자가 원본을 고쳤다면 그것은 단순 최신화가 아니라 선호, 관심사, 구조 감각이 바뀐 증거일 수 있다. 에이전트는 그 차이를 평탄화하지 말고, 무엇이 추가·삭제·강조되었는지 따로 읽어야 한다.

## 편집 판단으로 번역하기

이 README가 실제 hourly run에 주는 판단은 단순하다.

- 원본이 짧아도 대응 mapped 노트는 `왜 짧은지`, `어떤 경계를 세우는지`, `다른 노트와 어떤 역할 차이가 있는지`를 설명할 수 있다.
- 원본에 직접 손대고 싶은 충동이 생기면, 먼저 mapped 노트나 log에 해석을 남기는 것이 기본 경로다.
- 사용자가 source를 직접 고친 흔적이 있으면 최신 내용을 덮어쓰기보다 변화 전후의 선호 차이를 별도로 보존한다.
- 외부 링크가 많은 날에도 현재 선택한 source 노트의 주제 경계가 우선이며, 외부 자료는 보조 렌즈로만 쓴다.

## 다른 노드와의 연결
- [[Portfolio]]: 원본 내용이 실제 실행·운영 성과를 담는 루트 정체성 노드라면, 이 README는 그 원본을 다루는 절차적 정체성 노드다.
- [[CHANGELOG]]: 공개 표면의 유지 박동을 보여 주는 기록이다. 이 README는 그런 거친 운영 기록도 원본 흔적으로 보존해야 함을 정한다.
- [[Meta/Collection]]: 작은 미해결 질문을 탐색 큐로 남기는 노드다. 이 README의 source 보존 원칙이 있어야 Collection의 미완성 질문도 삭제되지 않는다.
- [[Communication/Document]]: 문서가 어떤 재진입 표면을 맡는지 다룬다. 이 README는 source 레이어의 재진입 표면 자체를 정의한다.

## 에이전트가 다음에 지켜야 할 체크포인트
- 대응 source 노트를 먼저 전체 읽고, 기존 mapped 노트를 전체 읽은 뒤에만 편집한다.
- 한 번의 hourly run에서는 mapped 노트 하나만 수정한다.
- 원본 파일을 더 깔끔하게 만들고 싶어도 source 본문은 기본적으로 건드리지 않는다.
- 외부 링크 자료는 현재 선택한 원본 노트의 주제 경계 안에서만 보조 맥락으로 쓴다.
- 원본이 직접 수정된 흔적이 있으면 `human-reviews/`에 비교본을 남기고 선호 변화로 읽는다.
