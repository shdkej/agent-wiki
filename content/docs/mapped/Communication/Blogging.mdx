# Blogging

- 원본: `source/shdkej-content/Communication/Blogging.md`
- mapped: `agent-wiki/content/docs/mapped/Communication/Blogging.mdx`
- 역할: 원본 만다라트 노드와 1:1 대응하는 관리 노트
- 상태: 안정 단계 · 기록에서 출판으로 넘어가는 편집 루프 기준 유지

## 핵심 논지
이 노트에서 블로깅은 글 수를 늘리는 일이 아니다. 흩어진 메모를 만다라트 구조 안에 붙잡아 두고 반복해서 퇴고한 뒤, 충분히 익은 재료만 독자용 글·사진·영상·지도·템플릿으로 꺼내는 발행 전환 시스템이다. 위키는 사고를 오래 붙드는 내부 자산이고, 블로그는 그 재료를 독자용 흐름으로 다시 편집한 외부 산출물이다.

## 만다라트 위치에서 읽는 해석
`Communication/Blogging`은 Communication 안에서 내부 지식이 외부 독자에게 도착하는 표면을 맡는다. [Communication/Document](/docs/mapped/Communication/Document)가 작업 기억을 다시 쓸 수 있는 문서군으로 고정하고 [Human/Readability](/docs/mapped/Human/Readability)가 독자에게 맞는 문장 밀도를 묻는다면, Blogging은 그 재료를 개인 잡지처럼 지속 가능한 발행 리듬으로 바꾸는 노드다.

따라서 이 노트는 블로그 글쓰기 팁 모음이 아니라 **내부 축적 -> 독자 진입면 설계 -> 반복 코너화 -> 반응과 재발견을 통한 환류**를 다루는 카테고리 노드로 읽어야 한다.

## 원문 흐름 요약
원문은 AI 결과물을 그대로 쓰지 않고, 문서 길이와 흐름을 제한하며, 만다라트 안에서 계속 퇴고하겠다는 기준으로 시작한다. 이어 노트가 흩어져 다시 쓰이지 않는 문제, 8개 큰 주제와 고정된 제목을 반복 개선하는 방식, 위키와 블로그의 차이, 독자 타깃과 문체의 필요성을 설명한다.

중반에는 주제 선정, Post 후보, AI 보조, 시리즈, 직접 겪은 경험, 제목, 잡지형 편집 키워드, 글 길이 기준, 블로그·유튜브·아이템 아이디어가 이어진다. 후반에는 단순함, 검색 유입 독자, 위키 블로그의 문제, 자료-정보-지식-지혜, Divio 문서 모델, 번역의 어투와 원문 맛, 영상 촬영·자막·B컷·지도, 이미지·사진·만화, 네이버 블로그 템플릿과 체험단 메모가 붙는다.

겉으로는 넓지만 중심은 같다. 원문은 `무엇을 더 많이 쓸까`보다 `내가 겪은 것과 쌓아 둔 것을 어떤 독자 경험으로 다시 자를까`를 계속 묻는다.

## 이 노트가 실제로 다루는 질문
- 왜 새 글을 계속 늘리기보다 제한된 카테고리를 반복 개선하려 하는가?
- 위키와 블로그는 무엇이 다르고, 각각 누구를 위한 공간인가?
- 검색해서 들어온 독자와 처음 접하는 독자를 같은 글 안에서 어떻게 감당할 것인가?
- favorite, Item List, 시리즈 후보는 단순 소재 목록인가, 반복 가능한 코너 후보인가?
- AI, 번역, 사진, 영상, 지도, 템플릿은 어디까지 보조하고 어디서 작성자의 시선을 지워 버리는가?
- 발행된 글은 어떻게 다시 기존 노트와 만다라트 구조를 호출하게 만드는가?

## 중심 모델: 내부 위키에서 발행 환류까지

| 단계 | 원문 단서 | 블로그 운영에서 맡는 역할 |
| --- | --- | --- |
| 축적 | 만다라트, 고정된 8가지 제목, 기존 글 업데이트, 계속 퇴고 | 파편 메모를 새 글로 흩뜨리지 않고 큰 관심사 안에 붙잡는다. |
| 선별 | 도움이 되는 나의 경험, 선택 이유, 인터넷에 널린 이야기 배제 | 일반론보다 직접 겪은 장면과 판단을 발행 후보로 올린다. |
| 진입면 | 한눈에 흐름, 독자 타깃, 검색 유입의 짧은 서론, 처음 접하는 독자의 맥락 | 독자가 어디서 들어오든 다음 문단으로 넘어갈 이유를 만든다. |
| 매체화 | 사진 설명, 번역 어투, 영상 컷, 지도, 네이버 블로그 템플릿 | 같은 경험을 글·이미지·영상·지도 중 어떤 표면으로 전달할지 고른다. |
| 환류 | 노트 리마인더, 기존 글 개선, 데이터가 충분해지면 글로 만들기 | 발행물이 끝이 아니라 기존 메모를 다시 호출하는 표지판이 되게 한다. |

이 모델로 보면 블로그의 완료 기준은 `게시했다`가 아니라 `내부 위키의 재료가 독자에게 읽히는 단위로 바뀌었고, 다시 다음 정리로 돌아올 수 있게 됐다`에 가깝다.

## 외부/실행 산출물 연결
`infinity/reports/research-21/2026-06-23T1500Z.html`의 `기록에서 출판으로` 조사는 이 노트의 블로깅을 `영감이 생기면 새 글을 쓰는 일`보다 `작은 기록 단위를 주제별로 재조립해 발행으로 바꾸는 편집 루프`로 더 선명하게 확인해 준다. 이 리포트가 정리한 `작은 단위 기록 → 주제별 조립 → 정리 과정에서 공백 검증 → 창작이 아니라 편집으로 변환` 흐름은, 원문의 `고정 된 8가지 제목의 글을 업데이트`, `데이터가 충분해지면 내용들을 정리해서 하나의 글로 만든다`, `위키는 내 것이고, 블로그는 독자의 것이다`를 실행 가능한 발행 모델로 압축한 형태다. 그래서 이 노트에서는 발행 직전마다 새 소재를 더 찾기보다, 이미 쌓인 메모가 어떤 주제 묶음으로 조립됐는지와 그 묶음이 독자용 글·사진·영상 중 어느 표면으로 가장 자연스럽게 변환되는지를 먼저 확인하는 게 맞다.

## 핵심 판단 축

| 축 | 보존할 판단 |
| --- | --- |
| 확장성 vs 제한 | 8x8 만다라트와 문서 길이 제한은 답답한 규칙이 아니라 깊이를 만들기 위한 상한선이다. |
| 내부 축적 vs 외부 발행 | 위키에는 파편과 미완성이 허용되지만, 블로그는 독자 타깃·문체·흐름이 정리되어야 한다. |
| 검색형 답변 vs 개인 잡지 | 검색 유입 글은 빠른 결론을 줘야 하지만, 전체 블로그는 미니멀 테크·데스크·자동화·생활 도구 같은 반복 세계관을 가져야 한다. |
| 일반론 vs 직접 경험 | 독자는 선택지 중 무엇을 왜 골랐는지, 실제로 어떤 시행착오를 겪었는지를 궁금해한다. |
| 자동화 편의 vs 작성자 의도 | AI, 번역, 템플릿은 초안을 돕지만, 최종 목소리와 판단 책임은 작성자에게 남아야 한다. |
| 자연스러움 vs 진행 표지 | 과한 조명·필터·연출은 경계하지만, 자막·B컷·사진 설명·지도처럼 독자의 이동을 돕는 표지는 적극적으로 쓴다. |

## 원문 고유 장면으로 남길 것

| 원문 장면 | 이 노드 안에서의 역할 |
| --- | --- |
| `AI를 통해 얻은 컨텐츠를 절대 그대로 가져오지 않는다` | 자동 생성물을 자기 언어와 경험으로 다시 통과시키는 기준이다. |
| `위키는 내 것이고, 블로그는 독자의 것이다` | 내부 기록과 외부 발행을 가르는 핵심 경계다. |
| `고정 된 8가지 제목의 글을 업데이트` | 새 글 생산보다 제한된 주제의 누적과 재검토를 선호한다는 신호다. |
| `하나의 잡지를 쓴다고 생각하면` | 블로그를 검색 답변 모음이 아니라 반복 코너와 편집 약속이 있는 매체로 본다. |
| `검색해서 들어갈 때는 서론을 굳이 안 읽고 싶다` | 독자 진입 경로별 첫 화면 설계가 필요하다는 단서다. |
| `desk setup`, `what i use`, `minimal travel setup`, `Item List` | 취향 나열이 아니라 시간이 지나도 반복 가능한 발행 포맷 후보에 가깝다. |
| 번역의 문장 길이·어투 고민 | 읽힘을 높이되 원작자의 의도와 맛을 지우지 않으려는 전달 윤리다. |
| 영상 5초 컷, 좌측 상단 자막, B컷, 지도 | 텍스트 밖 매체에서도 독자가 길을 잃지 않게 하는 진행 표지다. |
| 사진 10개와 3-4줄 주석, 일관성·가독성·나다움 | 템플릿은 작성 뼈대지만 시선이 사라지면 실패한다는 기준이다. |

## 매체별 발행 기준

| 매체 | 원문 기준 | 주의할 점 |

| 글 | 한눈에 흐름, 명확한 독자, 직접 겪은 경험, 너무 길지 않은 문서 | 개인 노트를 그대로 공개하면 독자용 산출물이 아니라 내부 덤프가 된다. |
| 번역 | 형태와 문장 길이를 존중하되 한국식 어투로 다듬기 | 읽힘만 좇아 원문의 단어 선택과 시대감을 지우지 않는다. |
| 영상 | 폰만 사용, 자연스러운 시선, 안정감, B컷, 좌측 상단 안내 자막 | 화려한 연출보다 주제와 진행 표지가 먼저다. |
| 이미지·만화 | 감정 표현, 한 문장 한 말풍선, 사진 위·설명 아래 | 장식보다 장면 이해를 돕는 배치가 중요하다. |
| 네이버 블로그 | 사진 10개, 3-4줄 주석, 담백함, 일관성·가독성·나다움 | 템플릿은 필요하지만 템플릿 냄새가 시선을 지우면 실패한다. |

## 인접 노드 연결
- [Communication/Document](/docs/mapped/Communication/Document): 내부 지식을 독자가 다시 움직일 수 있는 문서 표면으로 바꾸는 문제를 공유한다.
- [Human/Readability](/docs/mapped/Human/Readability): 한눈에 그려지는 글, 서론과 본론의 밀도, 독자별 설명량 조절과 직접 연결된다.
- [Communication/Feedback](/docs/mapped/Communication/Feedback): 발행 이후의 반응과 재검토가 다음 글 구조를 바꾸는 환류 루프를 맡는다.
- [blog/Note_Management](/docs/mapped/blog/Note_Management): 흩어진 메모를 다시 발견하고 발행 재료로 끌어올리는 문제의식이 같다.
- [blog/Digital_Content](/docs/mapped/blog/Digital_Content): 디지털 자산, 영상, 소장, 큐레이션 감각이 블로그의 발행 표면과 맞닿아 있다.
- [Integration/Tool](/docs/mapped/Integration/Tool): 앱·기어·자동화 소재를 기능 목록이 아니라 생활과 작업의 선택 기준으로 읽는 축을 공유한다.

## 후속 정리 기준
새 메모를 붙일 때는 먼저 아래 위치 중 하나에 놓는다.

| 새 메모 유형 | 먼저 붙일 위치 |
| --- | --- |
| 새 글감, favorite, Item List, 시리즈 후보 | `반복 코너 후보` |
| 제목, 서론, 글 길이, 독자 타깃, 검색 유입 고민 | `진입면` |
| AI 보조, 번역, 사진, 영상, 지도, 템플릿 | `매체별 발행 기준` |
| 노트 리마인더, 기존 글 개선, 위키와 블로그 경계 | `발행 환류` |
| 문서화, 가독성, 노트 관리, 도구 판단으로 더 잘 설명되는 항목 | 인접 노드로 링크하고 이 노드 안에서는 역할만 짧게 남긴다. |

## 드리프트 점검
대응 원본 `source/shdkej-content/Communication/Blogging.md`의 최근 git 이력은 초기 동기화에 머물러 이번 실행의 직접 사용자 수정본으로 보이지 않았다. `human-reviews/` 복사는 생략했다. 외부 링크 소스는 이번 압축과 직접 관련성이 낮아 사용하지 않았다.
