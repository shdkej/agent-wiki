# 지식 랩 운영 가설

이 랩의 목적은 사용자의 장문 노트를 원본 경계를 유지한 채 더 관리하기 쉬운 고품질 위키로 계속 재구성하는 것이다.

## 핵심 가설
- 원본 노트는 사용자의 원래 사고 방식을 보존한다.
- 에이전트 위키는 더 정돈되고 연결된 지식 레이어를 보존한다.
- 나중에 들어오는 인간 수정본은 보정 신호로 작동한다.
- 드리프트는 관찰 가능하고 되돌릴 수 있고 비교 가능하다면 허용할 수 있다.

## 성공 조건
에이전트는 범용적으로 더 똑똑해지는 것이 아니라, 사용자의 반복되는 니즈, 기준, 선호 작업 방식에 더 잘 맞아가야 한다.

## 위험
- 문체 드리프트
- 과도한 추상화
- 원본 근거를 벗어난 구조 발명
- 페이지 비대화

## 제어 장치
- 500줄 제한
- 카테고리 단위 작업
- 일일 요약
- 원본 기반 재생성 가능성

## 2026-06-08 이후 운영 전환

`source/shdkej-content/`는 사용자의 기본 사고 구조를 보존하는 안정 기준 소스로 남긴다. 1차 mapped 사이클이 충분히 돌았으므로, 이후의 지식 증분은 `source/external-links/`와 `infinity/` 산출물에서 더 많이 온다.

외부 링크는 세상에서 들어오는 새 재료이고, Infinity 산출물은 실제 작업을 통과한 판단의 압축본이다. agent-wiki는 이 둘을 단순 요약으로 보관하지 않고, 사용자의 장기 판단에 재사용 가능한 개념·비교·운영 원칙으로 승격해야 한다.

성공 기준은 문서 수 증가가 아니라, 외부 자료와 실행 결과가 사용자 고유의 지식 구조 안에서 다시 꺼내 쓸 수 있는 형태로 연결되는 것이다.
