# 지식 랩 운영 규칙

## 목표
사용자의 노트와 작업 방식을 점점 더 잘 반영하는 장기 운영형 LLM 위키를 만든다.

## 주요 개선 방향
- 탐구(Exploration): 각 노트 안에서 오래 남을 생각, 숨은 패턴, 긴장관계, 연결점, 미래에 다시 쓸 만한 지식을 발견한다.
- 정리(Cleaning): 가독성, 구조, 일관성, 중복 제어, 훑어보기 쉬움을 높이되 노트의 핵심 주제는 바꾸지 않는다.

## 기준 소스
- 안정 기준 소스: `source/shdkej-content/`
- 적극 참조 소스: `source/external-links/`
- 실행/판단 산출물: `infinity/`
- 컴파일 산출물: `agent-wiki/`
- 인간 수정/비교본: `human-reviews/`
- 운영 스키마: `schema/`

## 제약
1. 각 노트의 기본 구조와 주제를 유지한다.
2. 소스는 8x8 만다라트 구조로 본다. 상위 폴더는 대분류이고, 각 markdown 노트는 임의 문서가 아니라 하나의 카테고리 노드다.
3. 가능하면 각 원본 markdown 노트는 하나의 대응 위키 노트와 1:1로 매핑한다.
4. 생성·유지하는 모든 문서는 500줄 이하를 유지한다.
5. 기존 주제 경계 안에서만 명료화, 링크 보강, 구조 개선을 수행한다.
6. 사용자가 나중에 노트를 직접 고치면 `human-reviews/`에 반영하고, 에이전트 버전과 비교한다.
7. 차이는 무조건 덮어쓰지 말고 선호 변화로 문서화한다.
8. 시간별 작업은 한 번에 한 카테고리 또는 소수의 대응 노트에 집중한다.
9. `source/shdkej-content/` 기본 원본은 1차 사이클이 상당히 반영된 안정 기준으로 본다. 이후 반복 작업은 같은 원본을 계속 확장하기보다 외부 링크와 Infinity 산출물에서 새로 배운 판단을 관련 agent-wiki 페이지에 컴파일하는 데 더 많은 비중을 둔다.
10. `source/external-links/`는 사용자 원본 노트와 섞지 않는 별도 외부 소스 레이어로 유지하되, 2026-06-08 이후에는 제한적 참고 자료가 아니라 적극적인 위키 입력 후보로 본다. 단순 요약에 그치지 말고 관련 source/concept/synthesis/mapped/diary 페이지 중 어디에 축적할지 판단한다.
11. Infinity 산출물(`INTENTS.md`, archive 코멘트, intent 상세 파일, reports, `MARKETING_LEARNINGS.md`)은 실행 결과와 판단 기준이 응축된 보조 원천으로 본다. 완료된 산출물이 반복 가능한 원칙, 제품/마케팅 판단, 운영 교훈을 담고 있으면 agent-wiki의 syntheses/concepts/diary 또는 관련 mapped 노트에 반영한다.
12. 외부 링크나 Infinity 산출물이 존재한다고 해서 무조건 agent-wiki 본문에 넣지는 않는다. 관련성, 반복 가능성, 사용자 판단에 미치는 영향이 분명할 때만 승격하고, 원문/실행 로그의 경계와 출처를 명시한다.
13. 매일 사용자 요약에는 아래를 포함한다.
   - 어떤 카테고리를 탐구했는지
   - 어떤 페이지를 갱신했는지
   - 어떤 사용자 선호/패턴을 배웠는지
   - 어떤 드리프트나 이상 징후가 있었는지
14. 이 랩 안에서의 일반적인 위키 재작성에는 사용자 허가가 필요 없다.
15. 결과가 이상해지면 원본에서 같은 시스템을 다시 생성하고 변화 경로를 비교한다.
16. `source/` 아래 새 문서는 `schema/source.md`의 frontmatter, Source Boundary, 출처/상태/재사용 포인트 규칙을 따른다.
17. source 레이어는 raw material이고, 장기 판단·비교·개념화·질의 답변은 `agent-wiki/` 레이어에 컴파일한다. 좋은 답변이나 반복 가능한 판단은 채팅에만 두지 말고 wiki 또는 log에 누적한다.
18. schema 변경이 발생하면 `schema/`, `configs/agent-rules.md`, 필요한 하위 README가 서로 모순되지 않게 같은 흐름에서 갱신한다.

## 원하는 결과
- 사용자와 에이전트의 장기적 핏 향상
- 노트와 작업 이력을 바탕으로 한 사용자 니즈 예측력 향상
- 위임받은 작업의 결과 품질 향상
